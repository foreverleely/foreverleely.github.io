项目设置

新建工程导入 SDK 包

新建工程后,去极光推送网站下载对应iOS SDK包,将里面的lib文件夹复制到工程中。

build phases

在项目target下build phases导入框架
CFNetwork.framework
CoreFoundation.framework
CoreTelephony.framework
SystemConfiguration.framework
CoreGraphics.framework
Foundation.framework
UIKit.framework
Security.framework
Xcode7需要的是libz.tbd；Xcode7以下版本是libz.dylib
Adsupport.framework (获取IDFA需要；如果不使用IDFA，请不要添加)

build setting

在项目target下build setting下
设置 Search Paths 下的 User Header Search Paths 和 Library Search Paths，
比如SDK文件夹（默认为lib）与工程文件在同一级目录下，则都设置为”$(SRCROOT)/{静态库所在文件夹名称}”即可。

http 全局配置

在项目的 info.plist 中添加一个Key：NSAppTransportSecurity，类型为字典类型。
然后给它添加一个 NSAllowsArbitraryLoads，类型为布尔类型,并设为YES。

member center基本设置

在项目 target 下 general 的 identity 中选择好 team ,然后到 capabilities 打开 push notification ,Xcode会在我们的开发者账号下,
自动为我们创建当前应用的App ID(identifer),APNs证书(certificate),以及配置文件(provisioning profile);当然,也可以自己手动到 member center 创建。

push notification Development SSL Certificate

在 App IDs 目录下,找到当前应用的ID,选择 edit ,在 push notification 一栏下,选择 Development SSL Certificate 创建证书
,
按照提示生成证书,并上传到 member center;

下载APNs证书

在 member center 的 certificate 下,找到当前应用的APNs证书,选择下载到本地,在本地双击证书,证书会被保存到钥匙串中,
在钥匙串登录栏,证书目录下,找到刚刚添加进来的证书,右键选择导出(注意不要点开钥匙串下证书左侧的展开箭头,导出的文件格式为.p12格式,注意记好导出时设置的密码),

JPush 上创建应用

在 JPush 平台上创建应用,填写好相关参数,并上传刚刚生成的 .p12 格式文件,并填写文件导出时的密码,点击创建应用按钮。

消息推送测试

在 JPush 平台,自己账户下的应用列表下 https://www.jiguang.cn/app/list
,
点击推送按钮,进到发送通知页面,注意发送通知时的手机状态,有发送锁屏时的消息,也会发送打开应用时的消息。

错误汇总

按照官方文档基本不会出错,但是偶尔会有一些错误,比如,
错误一,真机调试(只能在真机下测试)时,可能会出现 console log 信息,提示 APNs证书配置或网络不好引起 SDK 运行失败,这个一般是网络问题;
错误二,出现 console log 信息,提示需要在 info.plist 下添加 remote notification ,我们只需要在工程 target capablities 下,打开
background modes ,并勾选 remote notification
