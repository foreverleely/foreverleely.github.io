1.我们分别在 objc 和 html 中定义好 registerHandler 和 callHandler事件；
2.html 中响应相关操作时需要通知objc这边时，就执行 html 的 callHandler，这时 objc 这边的 registerHandler 就会将响应结果返回给 html；
3.objc 中响应相关事件时（比如要求 html 页面执行某操作时）就执行 objc 的 callHandler，这时 html 这边的 registerHandler 就会将响应结果返回给 objc；
4.从2，3，我们看到了两对对应关系，而这两对对应关系是根据相同的 handlername 来实现相互匹配的，即2中的 handlername 一样，3中的 handlername 一样；
5.html 和 objc 之间实现通信的数据格式，一般选择 json;
6.html 效果还是不过 原生的效果;但是在编写方面 html 优于 objc。
